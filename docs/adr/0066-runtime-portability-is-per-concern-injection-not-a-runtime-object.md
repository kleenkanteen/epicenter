# 0066. Server runtime portability is per-concern injection chosen by whether an open standard exists, not a Runtime object

- **Status:** Accepted
- **Date:** 2026-06-24
- **Relates:** the in-flight spec `specs/20260623T234500-one-server-runtime-port-vs-per-owner-instance.md` (the grill and wave plan this harvests), [ADR-0054](0054-an-inference-backend-is-the-metered-gateway-or-a-custom-server.md) (the same config-not-code, deployment-chooses posture for inference backends), [ADR-0004](0004-trust-the-relay-reject-zero-knowledge.md) (why identity/secrets stay a shared plane, not per-instance), [ADR-0092](0092-identity-is-the-partition.md) (keeps the injection posture but collapses the ownership seam into the principal resolver)

## Context

`@epicenter/server` is one Hono library behind two Cloudflare deployables (`apps/api` personal, `apps/self-host` shared wiki). Two capabilities drove a portability push: the content-addressed blob store (a pure aws4fetch S3 client, no R2 binding) and the room relay (already behind the `Rooms`/`RoomUpdateLog`/`RoomSocket` contracts, with only a Cloudflare backend). Two tempting unifications appeared: a per-owner "instance" that hosts both capabilities, and a single `Runtime` god-object with co-equal `db`/`sessionStore`/`assets`/`rooms`/`afterResponse` legs handed to `createServerApp`. A first-principles grill rejected both. The instance is a deployment topology for a different product (hosting arbitrary per-owner code) and a false merge of two unlike lifecycles (a mutable single-writer live actor vs immutable owner-less bytes). The god-object overstates the work roughly fivefold by treating portable-standard concerns as if they were rooms, and it re-bundles exactly what the `mount*` and `resolveOrigin` design deliberately unbundled.

## Decision

Decide runtime portability per subsystem by one question: **is there an open standard both runtimes already speak?**

- **Road 1 (collapse to the standard).** If yes, depend on the standard and maintain no second backend. Object storage collapses to S3-over-HTTPS (blobs, done). SQL collapses to the Postgres wire via `pg`/drizzle (db: portable already; only connection *acquisition* differs). The Better Auth KV session cache is **deleted, not ported** (Postgres plus the JWE cookie cache is one path; this shipped).
- **Road 2 (inject a tiny per-concern contract).** If no, define the smallest Epicenter-owned contract and supply one backend per runtime, chosen at the `apps/*` edge. The only Road-2 subsystem is the room: a hibernating, single-writer, stateful actor has no open standard.

The seam is **per-concern injection** at the deployment edge (`resolveOrigin`, `connectDb`, `afterResponse`, `resolveRooms`), in the shape of the already-shipped `resolveOrigin`. There is **no single `Runtime` object**, and library code never reads `Cloudflare.Env` or imports `cloudflare:workers`. The Durable Object is the cloud's Road-2 binding of the room actor, **not the unit of deployment**; identity, billing, and the shared Postgres are a global plane, not per-instance state. A Bun/Node dev entry (`apps/api/server.ts` beside `worker/index.ts`, same Hono app) is the keystone second runtime that validates the seam; both entries are kept permanently.

## Consequences

- **Self-host can drop Cloudflare:** one binary plus a Postgres URL plus any S3 endpoint, no account required. The second runtime is **Bun specifically** (`bun:sqlite` is the built-in synchronous engine the room update log needs, `Bun.serve` the WebSocket host, `bun build --compile` the single-binary self-host artifact and Tauri sidecar); the earlier "Node / `better-sqlite3`" hedge is dropped as unbuilt optionality. The same Bun entry is what a Tauri shell embeds locally.
- **The library reads no Cloudflare-specific binding, by value OR by type.** Once the waves land, `db`, `afterResponse`, `rooms`, and `blobs` are injected per concern, and the library's env *type* is its own `ServerBindings` contract (an arktype schema), never `Cloudflare.Env`; only the deployment edge names a runtime, casting `env` to its own `Cloudflare.Env` to read a runtime-only binding (`HYPERDRIVE`, `ROOM`, `ASSETS_BUCKET`, `AUTUMN_SECRET_KEY`). The `cloudflare:workers` mock leaves the room tests. **Audit lesson:** the leak check must include ambient Cloudflare *globals*, not only `cloudflare:workers` imports and `c.env` reads. `WebSocketPair` (used on the auth-reject-over-WS path) slipped past an import-based audit and threw `ReferenceError` on Bun.
- **The cloud keeps per-room Durable Object sharding and hibernate-to-zero forever.** The collapse does not remove the DO; it makes the DO one binding. The per-room granularity and per-tenant scale-to-zero are exactly what multi-tenant cloud needs and what a single self-host process gets for free.
- **Two room backends must stay faithful to one `RoomCore`,** proven by running `RoomCore`'s tests against both. This is the standing maintenance cost Road 2 buys.
- **Dev-prod runtime skew is accepted and fenced:** a DO-only bug (hibernation, alarms, edge) will not surface in Bun dev, so `wrangler dev` / staging stays the fidelity gate before any deploy touching room behavior.
- **What this forecloses:** a single `Runtime` god-object (rejected as re-bundling), and the per-owner-instance product (a separate primitive that would still leave identity global). Refusing both is what keeps the seam explainable in one sentence.

## Refinements from the post-landing grill (2026-06-24)

An adversarial grill of the landed waves found the collapse was value-level done but type- and audit-level incomplete. The decision above did not change; these refine its consequences (some landed, some in flight):

- **The env contract is an arktype schema, validated at the Bun boot.** `ServerBindings` is now both the runtime schema and its inferred type (the `AuthUser` convention). The Bun entry runs it over `process.env` at startup (`ServerBindings.merge({ DATABASE_URL, … })(process.env)`), failing fast with one descriptive, aggregated error instead of an `as unknown as` cast. Cloudflare is **deliberately not** validated per-request: its bindings are deploy-gated and `wrangler types`-typed, and a Worker has no module-scope boot hook. Different env-acquisition guarantees earn different treatment (honest asymmetry).
- **`ASSETS_BUCKET` (an R2 object binding) left `ServerBindings`,** because a `process.env`-validatable schema holds only strings. The doomed Cloudflare-only assets route read it via a local `AssetsEnv` cast.
- **Assets is now folded into the portable blob store, so the library names zero Cloudflare object types — by value OR by type (2026-06-24).** The Cloudflare-only assets route, its `AssetsEnv` cast, the `asset` table, the `ASSETS_BUCKET` R2 binding (both deployables), the `client.assets` SDK surface, and the asset-table-coupled storage-billing wiring were all deleted; nothing consumed the assets route, so it was removed outright rather than migrated (blob spec slice 6). `ASSETS_BUCKET` was the last Cloudflare object binding the library named; `grep -rn "ASSETS_BUCKET\|R2Bucket" packages/server/src` is now empty. The only Cloudflare object types left in the tree are runtime-injected at the deployment edge (`ROOM` Durable Object, `HYPERDRIVE`) and the deployment-owned `ASSETS` static-assets binding (dashboard SPA), none of which the library names.
- **WS-upgrade auth rejection is a per-runtime capability.** Cloudflare mints a `WebSocketPair` and closes it `4401`; Bun cannot mint a detached socket, so the faithful path upgrades through the room backend and closes in the open handler (`Rooms.rejectUpgrade`). A `typeof WebSocketPair` capability-detect with an HTTP fallback is the floor that stops the crash; the room-backend reject is the faithful ceiling and the chosen direction.
- **One `RoomCore` conformance suite, not two drifting backend copies.** The connection-lifetime and liveness ping/pong invariants moved into `RoomCore` so both backends inherit them; the shared behaviors (presence debounce, the 4401 path, dispatch `RecipientOffline`, the compaction cap) should be exercised once against the core rather than partially re-tested per backend.
- **The runtime-port triple groups into a `RuntimeAdapter`; identity is a separate axis (2026-06-24).** The per-concern hooks first shipped as six flat options on `createServerApp`. A greenfield grill found two of the six framings were dishonest. The three runtime concerns (`connectDb`, `afterResponse`, `resolveRooms`) are never chosen independently — a per-request `pg.Client` over Hyperdrive, `waitUntil`, and the Durable Object are all the Cloudflare runtime, and both Cloudflare deployables wrote the identical triple verbatim — while the other three (`resolveOrigin`, `resolveTrustedOrigins`, `cookieDomain`) vary per *deployment*, not per runtime (apps/api passes the same identity on Workers and Bun, and `resolveTrustedOrigins` is byte-identical across them). So the triple now travels as one `RuntimeAdapter` value built at the edge (`cloudflare()` for both Workers deployables; an inline literal on Bun, which has a single producer), and `createServerApp` takes two named axes: `{ runtime, identity }`. This does **not** reintroduce the rejected god-object: that object's fault was carrying *portable* concerns (db driver, session store, assets) as co-equal legs and overstating the port fivefold; those collapsed to standards (Road 1), and the `RuntimeAdapter` holds only the genuinely co-selected non-portable triple. The earlier framing that called `resolveOrigin` "one of the per-concern runtime-port hooks" was the tell: it is a deployment-identity hook, not a runtime concern. `connectHyperdriveDb` and `createDurableObjectRooms` left the public barrel, now internal to `cloudflare()` (`runtime/cloudflare.ts`).

(Amended 2026-07-01: the `cookieDomain` deployment-identity hook was deleted when sessions went host-only, `SameSite=Lax` on the API origin, per [ADR-0079](0079-whispering-authenticates-with-an-oauth-bearer-on-every-surface.md); the cross-subdomain cookie knob it configured no longer exists. The other deployment-identity hooks (`resolveOrigin`, `resolveTrustedOrigins`) stand unchanged.)
